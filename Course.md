## Learning Philosophy

### Build-Your-Own-X
You don't truly understand something until you've built it from scratch. Throughout this course, you'll recreate fundamental tools: array methods, promises, hooks, query builders, and more. This approach builds deep understanding that tutorials cannot provide.

### No Solutions Provided
Labs describe the problemâ€”you discover the solution. This mirrors real-world development where Stack Overflow won't have your exact answer. Struggle is part of learning; embrace it.

### Deploy Early, Deploy Often
Five deployment checkpoints ensure you're building real, working softwareâ€”not just completing exercises. Employers want to see live projects.

### One Thing at a Time
Master each concept before moving on. Resist the urge to jump ahead. Foundations matter more than frameworks.

## Core Principles

### 1. Type All Code Manually
Never copy-paste. Typing builds muscle memory and forces you to read every character.

### 2. Struggle Before Searching
When stuck, set a timer for 15-30 minutes. Try different approaches. Only then search for help. Use the [When You're Stuck](guides/when-youre-stuck.md) guide.

### 3. Build, Don't Just Learn
Every week produces working code. Reading about programming isn't programming.

### 4. Master Before Moving
Don't rush to the next module. Shaky foundations create shaky applications. Complete the Self-Assessment Checkpoint at the end of each module before moving on.

### 5. Commit Everything
Use Git from day one. Small, frequent commits. Your commit history tells your story.
